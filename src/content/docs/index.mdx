## What is it?

The **Interaction System** is a Unity UPM package that gives your game a clean, extensible layer for player ↔ world interactions. It is built around three core ideas:

- **Camera-agnostic** — works with any camera setup; you provide the camera, the system handles the raycast.
- **Input-agnostic** — you call `HandleInteraction()` from your input code; no hard dependency on any input system.
- **Action-driven** — every interactable object exposes one or more `IInteractableAction` implementations, which you can mix freely in three styles.

  </Card>
  <Card title="Inspector-friendly" icon="setting">
    `[SerializeField]` fields, `[RequireComponent]` guards, editor menu helpers (`Create > FireSoftworks > Interaction`) and ScriptableObject configuration out of the box.
  </Card>
  <Card title="No external dependencies" icon="approve-check-circle">
    Zero coupling to inventory, UI, or physics systems. Drop it into any project and integrate at the event layer.
  </Card>
